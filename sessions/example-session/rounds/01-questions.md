# Round 1: Foundation

## Questions

### Q1.1: What type of application?

Is this a single-page app (SPA), server-rendered, or mobile app?
This affects which auth approach works best.

### Q1.2: User base

- How many users do you expect?
- Do you need social login (Google, GitHub)?
- Any compliance requirements (SOC2, GDPR)?

### Q1.3: Session duration

How long should users stay logged in?
- Hours (banking app)
- Days (productivity tool)
- Weeks (social media)

## Summary

Gathering requirements to recommend the right auth approach.
